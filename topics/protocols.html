Communication Protocols

UART, SPI, I²C and CAN — the four serial languages every embedded engineer must speak.

01. What it is

Embedded systems rarely talk in parallel any more — pin counts and EMI rule it out. Instead they use a small family of serial protocols, each with its own trade-offs in speed, wires, addressing and noise immunity.

  • UART — async, point-to-point, 2 wires, ~5 Mbps max.
  • SPI — sync, full-duplex, 4 wires, 80 MHz+ — fast, simple, no addressing.
  • I²C — sync, half-duplex, 2 wires, 100 kHz to 3.4 MHz — many devices, one bus.
  • CAN — sync, differential, multi-master, 1 Mbps (8 Mbps for CAN-FD), bullet-proof in cars.

02. How it works

Each protocol defines a frame format at the bit level. UART frames have a start bit, 5–9 data bits, optional parity, and 1–2 stop bits. SPI just shifts bits while CS is low. I²C frames begin with a START condition, an address byte, a R/W bit, and ACK/NACK from the addressed slave. CAN frames carry an 11- or 29-bit arbitration ID; the lowest ID always wins the bus without collisions.

Picking a protocol is mostly about how many devices you have, how far apart they are, and whether the link can be hot-plugged or live in an electrically noisy environment.

04. Real-world examples

  • USB-to-UART bridge (FT232, CP2102) — every Arduino "Serial Monitor" goes through one.
  • SD-card SPI mode — every microcontroller can talk to an SD card with just SPI + CS.
  • BME280 environmental sensor — 7-bit I²C address 0x76, supports both I²C and SPI.
  • OBD-II in cars — every passenger car since 2008 exposes engine telemetry over CAN.
  • Pixhawk autopilots — talk to ESCs over UART, IMUs over SPI, GPS over I²C, and other companion computers over CAN.

05. Code snippet

C — I²C sensor read 
#define BME280_ADDR 0x76
#define REG_TEMP    0xFA

uint8_t reg = REG_TEMP, raw[3];

HAL_I2C_Master_Transmit(&hi2c1, BME280_ADDR << 1,
                          &reg, 1, 100);
HAL_I2C_Master_Receive (&hi2c1, (BME280_ADDR << 1) | 1,
                          raw, 3, 100);

int32_t rawT = (raw[0] << 12) | (raw[1] << 4) | (raw[2] >> 4);

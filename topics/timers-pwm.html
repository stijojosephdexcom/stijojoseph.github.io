Timers & PWM

A hardware timer is a free-running counter — feed it the right comparison and it generates everything from precise delays to motor PWM.

01. What it is

An MCU timer is a counter that increments on every tick of its clock. Configurable prescaler and auto-reload (ARR) values let it count from 0 to N at any frequency. A compare register (CCR) lets you fire an event — toggling a pin, raising an interrupt, or triggering a DMA — when the counter passes a chosen value.

PWM (Pulse Width Modulation) is the simplest application: a square wave whose duty cycle (percent ON) is determined by the ratio CCR/ARR. PWM is how you dim LEDs, control servos, drive H-bridge motors and synthesise audio.

02. How it works

A typical PWM channel is configured as:

  1. Pick the timer clock (e.g. APB1 at 84 MHz).
  2. Set the prescaler so the counter ticks at a useful rate (e.g. ÷84 → 1 MHz, 1 µs per count).
  3. Set ARR to define the period (e.g. 20 000 → 50 Hz for servos).
  4. Set CCR to define the pulse width (e.g. 1500 → 1.5 ms = servo at 90°).
  5. Configure the output mode as PWM mode 1 (active until CCR, inactive after).

Input capture is the inverse: latch the counter value when an external signal edge arrives, letting you measure pulse widths or frequencies.

04. Real-world examples

  • RC servo control — 50 Hz period, 1–2 ms pulse for 0–180° angle.
  • LED dimming — 1 kHz PWM at variable duty cycle is invisible to the eye.
  • BLDC motor commutation — three complementary PWM pairs with dead-time insertion.
  • DC-DC buck regulator — high-frequency PWM (≥100 kHz) drives an inductor; the duty controls output voltage.
  • Frequency measurement — input capture latches the timer count on each rising edge.

05. Code snippet

C — STM32 PWM init 
/* TIM3 channel 1 on PA6 — 1 kHz PWM, 50% duty */
htim3.Instance               = TIM3;
htim3.Init.Prescaler         = 83;       /* 84 MHz / 84 = 1 MHz */
htim3.Init.CounterMode       = TIM_COUNTERMODE_UP;
htim3.Init.Period            = 999;      /* 1 MHz / 1000 = 1 kHz */
htim3.Init.ClockDivision     = TIM_CLOCKDIVISION_DIV1;
HAL_TIM_PWM_Init(&htim3);

TIM_OC_InitTypeDef oc = {0};
oc.OCMode     = TIM_OCMODE_PWM1;
oc.Pulse      = 500;             /* 50% of 1000 */
oc.OCPolarity = TIM_OCPOLARITY_HIGH;
HAL_TIM_PWM_ConfigChannel(&htim3, &oc, TIM_CHANNEL_1);
HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_1);

/* Change duty later */
__HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_1, 750); /* 75% */

ADC / DAC

ADCs digitise reality, DACs reconstruct it. Sample rate and bit depth set the fidelity.

01. What it is

An ADC (Analog-to-Digital Converter) samples a continuous voltage at fixed intervals and outputs an integer code proportional to the input. A DAC (Digital-to-Analog Converter) does the inverse, turning integer codes back into voltages.

Two specs dominate:

  • Sample rate (Hz) — how often you sample. By Nyquist's theorem you must sample at least twice the highest frequency you want to capture.
  • Bit depth (N bits) — how finely each sample is quantised. An N-bit ADC has 2N possible output codes; the resulting quantisation noise is your minimum error floor.

02. How it works

Most MCU ADCs are SAR (Successive Approximation Register) converters — they binary-search the input voltage in N clock cycles for an N-bit conversion. Common resolutions are 10, 12 or 16 bits.

Most DACs in MCUs are R-2R ladder or delta-sigma designs. Audio applications use 16/24-bit delta-sigma DACs with built-in oversampling and reconstruction filtering.

Practical pitfalls: input impedance must be low enough to charge the ADC's sample-and-hold capacitor; reference voltage drift directly translates to measurement error; long traces pick up noise that masks low-amplitude signals.

04. Real-world examples

  • Audio recording (44.1 kHz · 16 bit) — CD quality, the historical reference.
  • Digital oscilloscope — 1 GS/s · 8 bit ADCs are typical for entry-level scopes.
  • BLDC motor current sensing — 12 bit ADCs synchronised to PWM for FOC algorithms.
  • BLE pulse oximeter — 14–16 bit ADCs sample the photodiode at 100–1000 Hz.
  • Audio DAC for class-D amplifier — delta-sigma 24 bit, oversampled to 384 kHz.

05. Code snippet

C — STM32 ADC + DMA 
/* 12-bit single channel, DMA into a buffer */
uint16_t samples[256];

hadc1.Init.Resolution = ADC_RESOLUTION_12B;
hadc1.Init.ScanConvMode = DISABLE;
hadc1.Init.ContinuousConvMode = ENABLE;
hadc1.Init.DMAContinuousRequests = ENABLE;
HAL_ADC_Init(&hadc1);

HAL_ADC_Start_DMA(&hadc1, (uint32_t*)samples, 256);

void HAL_ADC_ConvCpltCallback(ADC_HandleTypeDef *h) {
    uint32_t sum = 0;
    for (int i = 0; i < 256; i++) sum += samples[i];
    float volts = (sum / 256.0f) * 3.3f / 4095.0f;
    processVoltage(volts);
}

Watchdog Timer (WDT)

A watchdog is a countdown bomb. Pet it in time, you live; miss a deadline, the MCU resets — guaranteed.

01. What it is

A watchdog timer is an independent counter inside the MCU that decrements (or counts up to a reload value) on its own clock. If your firmware doesn't periodically reload ("kick" or "pet") the watchdog within a configured window, the watchdog asserts a system reset — the MCU reboots and starts over.

That single-purpose bit of silicon is what stops your toaster from staying on for hours when its firmware crashes.

02. How it works

On STM32 there are two watchdogs:

  • IWDG (Independent Watchdog) — clocked by the LSI (~32 kHz), runs even if the main PLL stops. Once started it cannot be disabled. Timeout from a few ms to several seconds.
  • WWDG (Window Watchdog) — also detects "petted too early", catching runaway loops that would otherwise look healthy.

Best practice: kick the watchdog in one well-known place in your main loop. Don't sprinkle WDT_kick() all over the codebase — that defeats the purpose, because a stuck branch could still kick.

04. Real-world examples

  • Industrial controllers (PLCs) — required for IEC 61508 SIL-2 compliance.
  • Automotive ECUs — paired with an external safety chip that watches the MCU's heartbeat.
  • Set-top boxes — quietly reboot themselves overnight if a network stack hangs.
  • Pacemakers and defibrillators — multiple redundant watchdogs are a regulatory requirement.

05. Code snippet

C — STM32 IWDG init 
/* IWDG: LSI (~32 kHz) / 64 prescaler = 500 Hz tick */
/* Reload 2000 -> 4 second timeout */
hiwdg.Instance       = IWDG;
hiwdg.Init.Prescaler = IWDG_PRESCALER_64;
hiwdg.Init.Reload    = 2000;
HAL_IWDG_Init(&hiwdg);

void main_loop(void) {
    while (1) {
        doWork();
        HAL_IWDG_Refresh(&hiwdg); /* pet the dog */
    }
}

/* Diagnose reset cause on boot */
if (__HAL_RCC_GET_FLAG(RCC_FLAG_IWDGRST)) {
    log("watchdog reset");
    __HAL_RCC_CLEAR_RESET_FLAGS();
}

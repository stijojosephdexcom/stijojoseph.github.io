Memory Architecture

Flash holds your code, SRAM holds your data — and the stack and heap grow toward each other in a 64 KB cage.

01. What it is

Embedded MCUs typically use a Harvard-style bus matrix: separate paths for instruction fetch (Flash) and data access (SRAM). The CPU has a single 32-bit address space, but each region maps to different physical hardware with very different characteristics.

  • Flash — non-volatile, holds .text (code) and .rodata (constants). Slow to write (sector-erase first).
  • SRAM — volatile, holds .data (initialised globals), .bss (zeroed globals), the stack and the heap.
  • EEPROM / NV storage — byte-erasable for config and calibration.
  • Peripheral registers — memory-mapped I/O at fixed addresses.

02. How it works

The linker script tells the linker which physical regions exist and which sections go where. At reset the startup code copies .data from Flash to SRAM, zeroes .bss, sets the stack pointer to the top of SRAM, and jumps to main().

The stack grows downward from the top of SRAM as functions are called; the heap grows upward from the end of .bss as malloc() requests blocks. If they meet, you have a stack–heap collision and undefined behaviour. On a 32 KB SRAM device, leaving 4 KB of safety margin is common.

04. Real-world examples

  • STM32F4 vector table at 0x08000000 — first word is initial MSP (stack pointer), second is reset handler.
  • Embedded C bare-metal startup — copies .data from Flash _etext to _sdata; zeroes .bss.
  • Stack-canary checks — write a known pattern at the bottom of stack and verify periodically.
  • FreeRTOS heap_4.c — coalescing best-fit allocator tuned for fragmentation.

05. Code snippet

GNU LD — linker script excerpt 
MEMORY
{
  FLASH (rx) : ORIGIN = 0x08000000, LENGTH = 256K
  SRAM  (rwx): ORIGIN = 0x20000000, LENGTH = 64K
}

SECTIONS
{
  .text : {
    KEEP(*(.isr_vector))
    *(.text*)
    *(.rodata*)
  } > FLASH

  .data : {
    _sdata = .;
    *(.data*)
    _edata = .;
  } > SRAM AT> FLASH

  .bss : {
    _sbss = .;
    *(.bss*)
    _ebss = .;
  } > SRAM

  ._user_heap_stack : {
    . = ALIGN(8);
    PROVIDE ( end = . );
    . = . + 0x4000; /* heap */
    . = . + 0x2000; /* stack */
  } > SRAM
}

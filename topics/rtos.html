Real-Time Operating System (RTOS)

An RTOS lets your firmware do many things at once with deterministic timing — preempt, prioritise, communicate.

01. What it is

An RTOS is a small kernel that gives an MCU the illusion of running many tasks in parallel. Unlike a desktop OS that optimises for throughput, an RTOS optimises for determinism: the highest-priority ready task always gets the CPU within a known, bounded time.

That property makes RTOSes essential wherever a missed deadline is dangerous — motor control, medical devices, avionics, automotive ECUs.

02. How it works

The kernel maintains a ready queue of tasks ordered by priority. Whenever an event occurs (a tick interrupt fires, an interrupt unblocks a task, or the running task yields), the scheduler runs and picks the highest-priority ready task. If that task is different from the current one, a context switch saves the CPU registers of the old task to its stack and restores those of the new task.

  • Tasks have their own stack, priority and state (Ready / Running / Blocked / Suspended).
  • Queues, semaphores, mutexes and event groups let tasks communicate safely.
  • Tick interrupt drives time-based delays (vTaskDelay) and the round-robin slice for equal-priority tasks.
  • Priority inheritance on mutexes prevents priority inversion.

04. Real-world examples

  • FreeRTOS — the most-deployed embedded RTOS, MIT-licensed, runs on >40 architectures.
  • Zephyr — Linux-Foundation-hosted RTOS with built-in networking, BLE, USB, file systems.
  • ThreadX (Azure RTOS) — small footprint, certified for safety-critical use.
  • RTEMS — used in spacecraft including the Mars rovers and ESA missions.
  • NuttX — POSIX-like API, used in PX4 drone autopilots.

05. Code snippet

C — FreeRTOS task 
#include "FreeRTOS.h"
#include "task.h"
#include "queue.h"

static QueueHandle_t xTempQueue;

void vSensorTask(void *p) {
    for (;;) {
        float t = readTemperature();
        xQueueSend(xTempQueue, &t, 0);
        vTaskDelay(pdMS_TO_TICKS(100));
    }
}

void vDisplayTask(void *p) {
    float t;
    for (;;) {
        if (xQueueReceive(xTempQueue, &t, portMAX_DELAY) == pdTRUE)
            drawTemperature(t);
    }
}

int main(void) {
    xTempQueue = xQueueCreate(8, sizeof(float));
    xTaskCreate(vSensorTask,  "sensor",  256, NULL, 3, NULL);
    xTaskCreate(vDisplayTask, "display", 256, NULL, 2, NULL);
    vTaskStartScheduler();
    for (;;);
}

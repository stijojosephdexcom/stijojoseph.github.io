Power Management

Modern MCUs can go from 100 mA in run mode to under 1 µA in standby — that is the difference between a 10-day battery life and a 10-year one.

01. What it is

An MCU spends most of its life waiting. Power management is the art of making the CPU and its peripherals consume nothing while waiting, then waking up fast enough to handle the event when it finally occurs.

Every modern MCU offers a hierarchy of low-power modes, trading current draw against wake-up latency, retained state, and which wake-up sources are still alive.

02. How it works

Typical modes on a Cortex-M-class MCU:

  • Run — full speed, all clocks alive. Tens of mA.
  • Sleep — CPU clock gated; peripherals run. Wakes on any IRQ. Few mA.
  • Stop / Deep Sleep — main PLL off, SRAM retained, only LSI/LSE clocks. Few µA.
  • Standby — domain off; only RTC, wake-up pins and backup registers survive. Sub-µA.

Picking a mode is a balance: how often must the system wake, how fast must it wake, and does state need to persist?

04. Real-world examples

  • Apple AirTag — coin cell + nRF52, mostly in deep sleep, wakes briefly to advertise BLE. ≈1 year life.
  • LoRaWAN smart-meter — STM32L0 wakes once per hour to transmit a 30-byte packet. 10 year battery.
  • BLE medical wearable — sleeps between 7.5 ms BLE intervals, runs ECG sampling for 60 ms.
  • Industrial wireless sensor — RTC wakes the MCU every 5 minutes to read a temperature probe.

05. Code snippet

C — STM32L4 STOP2 mode 
/* Configure RTC wakeup every 60 s */
HAL_RTCEx_SetWakeUpTimer_IT(&hrtc, 60,
    RTC_WAKEUPCLOCK_CK_SPRE_16BITS);

while (1) {
    doMeasurement();
    transmit();
    /* Now sleep until the next RTC tick */
    HAL_SuspendTick();
    HAL_PWREx_EnterSTOP2Mode(PWR_STOPENTRY_WFI);
    /* Resumes here on wakeup */
    SystemClock_Config();
    HAL_ResumeTick();
}

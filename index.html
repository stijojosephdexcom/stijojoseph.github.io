Core Concepts

The building blocks of every embedded system

🧠
Microcontroller (MCU)
An MCU integrates CPU, RAM, Flash, and peripherals on a single chip. Popular families include ARM Cortex-M, AVR, PIC, and ESP32.
ARM Cortex-MAVRESP32
⏱️
Real-Time OS (RTOS)
An RTOS provides deterministic task scheduling, prioritisation, and inter-task communication via queues, semaphores and mutexes.
FreeRTOSZephyrThreadX
⚡
Interrupts & ISR
Interrupts allow the CPU to respond immediately to hardware events, pausing normal execution and calling an Interrupt Service Routine (ISR).
NVICISRPriority
💾
Memory Architecture
Embedded systems use Flash (code), SRAM (runtime data), EEPROM (persistent config), and registers. Harvard vs. von Neumann architecture matters here.
FlashSRAMEEPROM
🔄
DMA (Direct Memory Access)
DMA controllers transfer data between memory and peripherals without CPU involvement, greatly reducing latency and power consumption.
DMA ChannelsBurst ModeCircular
🕑
Timers & PWM
Hardware timers generate precise delays, measure frequencies, and produce PWM signals for motor control, LED dimming, and more.
PWMInput CaptureOutput Compare
📡
Communication Protocols
SPI, I²C, UART, CAN, USB — each protocol has trade-offs in speed, pin count, bus topology, and error detection.
SPII²CUARTCAN
🔋
Power Management
Sleep modes (sleep, deep sleep, stop, standby) drastically reduce current draw. Wake sources include RTC, GPIO, UART, and ADC threshold.
Sleep ModesLDODCDC
🛡️
Watchdog Timer (WDT)
The WDT resets the MCU if software fails to "pet" it within a timeout, guarding against software hangs in safety-critical systems.
IWDGWWDGTimeout
🔧
Bootloader
A bootloader runs before the main application, validating firmware integrity and enabling field updates (FOTA) over UART, USB, or wireless.
DFUFOTACRC
📊
ADC / DAC
Analogue-to-Digital (ADC) and Digital-to-Analogue (DAC) converters bridge the analogue world of sensors and actuators with digital processing.
SAR ADCSigma-DeltaResolution
🔍
Debugging & JTAG
JTAG and SWD interfaces allow a debugger to halt execution, inspect registers, set breakpoints, and trace instruction flow in real time.
JTAGSWDOpenOCD

Code Examples

Real-world snippets from popular embedded frameworks

C — FreeRTOS 
#include "FreeRTOS.h"
#include "task.h"

/* LED blink task */
void vLedTask(void *pvParams) {
    const TickType_t xDelay = pdMS_TO_TICKS(500);
    for (;;) {
        GPIO_TogglePin(LED_PORT, LED_PIN);
        vTaskDelay(xDelay);
    }
}

/* Sensor read task — higher priority */
void vSensorTask(void *pvParams) {
    for (;;) {
        float temp = readTemperature();
        xQueueSend(xTempQueue, &temp, portMAX_DELAY);
        vTaskDelay(pdMS_TO_TICKS(100));
    }
}

int main(void) {
    xTaskCreate(vLedTask,    "LED",    128, NULL, 1, NULL);
    xTaskCreate(vSensorTask, "Sensor", 256, NULL, 2, NULL);
    vTaskStartScheduler();
    for (;;); /* should never reach */
}

FreeRTOS Multi-Tasking

FreeRTOS is the world's most popular RTOS for microcontrollers, used on billions of devices. It provides preemptive or cooperative task scheduling.

  • Tasks are independent functions with their own stack
  • Higher-priority tasks preempt lower-priority ones
  • Queues pass data safely between tasks
  • vTaskDelay() yields the CPU for a specified tick count
  • pdMS_TO_TICKS() converts ms to scheduler ticks
C — STM32 HAL 
/* Configure EXTI line on PA0 */
HAL_NVIC_SetPriority(EXTI0_IRQn, 2, 0);
HAL_NVIC_EnableIRQ(EXTI0_IRQn);

/* ISR — keep it SHORT and FAST */
void EXTI0_IRQHandler(void) {
    if (__HAL_GPIO_EXTI_GET_IT(GPIO_PIN_0)) {
        buttonPressFlag = 1; /* set flag */
        __HAL_GPIO_EXTI_CLEAR_IT(GPIO_PIN_0);
    }
}

/* Main loop checks flag (deferred processing) */
void processButton(void) {
    if (buttonPressFlag) {
        buttonPressFlag = 0;
        toggleLED();
    }
}

External Interrupts (EXTI)

Interrupt-driven GPIO allows the MCU to react to button presses, sensor signals, or communication events without polling.

  • NVIC priority 0 = highest (pre-emptive)
  • ISRs must be short — avoid blocking calls
  • Use volatile flags for shared state
  • Deferred processing keeps ISR latency low
  • Always clear the interrupt flag to avoid re-entry
C — SPI HAL 
/* SPI full-duplex transmit/receive */
uint8_t txBuf[4] = {0x02, 0x00, 0x10, 0xFF};
uint8_t rxBuf[4] = {0};

/* Assert chip-select (active low) */
HAL_GPIO_WritePin(CS_PORT, CS_PIN, GPIO_PIN_RESET);

HAL_StatusTypeDef status =
    HAL_SPI_TransmitReceive(
        &hspi1, txBuf, rxBuf, 4, 10);

/* De-assert CS */
HAL_GPIO_WritePin(CS_PORT, CS_PIN, GPIO_PIN_SET);

if (status == HAL_OK) {
    processResponse(rxBuf, 4);
} else {
    handleSPIError(status);
}

SPI — Serial Peripheral Interface

SPI is a synchronous, full-duplex, master-slave protocol ideal for high-speed peripherals like flash memory, ADCs, and displays.

  • 4-wire bus: SCK, MOSI, MISO, CS
  • Speeds up to 80+ MHz on modern MCUs
  • CS line selects one of multiple slaves
  • Four modes (CPOL/CPHA) define clock polarity
  • DMA-based transfers offload the CPU entirely
C — I²C / BME280 
#define BME280_ADDR  0x76
#define REG_TEMP_MSB 0xFA

/* Write register address then read data */
uint8_t regAddr = REG_TEMP_MSB;
uint8_t rawData[3];

HAL_I2C_Master_Transmit(
    &hi2c1,
    BME280_ADDR << 1,
    ®Addr, 1, 10);

HAL_I2C_Master_Receive(
    &hi2c1,
    (BME280_ADDR << 1) | 0x01,
    rawData, 3, 10);

/* Combine 20-bit temperature raw value */
int32_t raw_temp =
    ((rawData[0] << 12) |
     (rawData[1] << 4)  |
     (rawData[2] >> 4));
float temp_c = bme280_compensate_temp(raw_temp);

I²C — Inter-Integrated Circuit

I²C is a 2-wire multi-master bus perfect for sensors and low-speed peripherals. Each device has a 7- or 10-bit address.

  • Only 2 wires: SDA (data) + SCL (clock)
  • Up to 127 devices on one bus
  • Standard mode 100 kHz, Fast 400 kHz, HS 3.4 MHz
  • Address shifted left by 1, LSB = R/W bit
  • Requires pull-up resistors on both lines
C — PWM / Servo 
/* TIM2 at 50 Hz (20 ms period) for servo */
HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_1);

/* Servo: 1ms = 0°, 1.5ms = 90°, 2ms = 180° */
void setServoAngle(uint8_t degrees) {
    /* Map 0-180° → 500-2500 µs pulse */
    uint32_t pulse =
        500 + (((uint32_t)degrees * 2000) / 180);
    /* CCR = pulse * (timer_freq / 1,000,000) */
    __HAL_TIM_SET_COMPARE(&htim2,
        TIM_CHANNEL_1, pulse);
}

/* LED dimming via duty cycle */
void setLEDBrightness(uint8_t pct) {
    uint32_t arr = __HAL_TIM_GET_AUTORELOAD(&htim3);
    __HAL_TIM_SET_COMPARE(&htim3,
        TIM_CHANNEL_2, (arr * pct) / 100);
}

PWM — Pulse Width Modulation

PWM generates a digital square wave with a variable duty cycle, allowing digital control of analogue quantities like brightness and motor speed.

  • Duty cycle = ON time / Period × 100%
  • Frequency set by ARR register (auto-reload)
  • Compare value (CCR) sets pulse width
  • Servos expect 50 Hz with 1–2 ms pulses
  • Complementary outputs for H-bridge motor drivers

#define RX_BUF_SIZE 256
volatile uint8_t rxBuf[RX_BUF_SIZE];

/* Circular DMA — never stops, CPU untouched */
HAL_UART_Receive_DMA(
    &huart1, (uint8_t*)rxBuf, RX_BUF_SIZE);

/* Idle-line detection triggers callback */
__HAL_UART_ENABLE_IT(&huart1, UART_IT_IDLE);

void HAL_UART_RxCpltCallback(UART_HandleTypeDef *h){
    if (h->Instance == USART1) {
        uint16_t pos = RX_BUF_SIZE -
            __HAL_DMA_GET_COUNTER(h->hdmarx);
        processPacket((uint8_t*)rxBuf, pos);
    }
}

/* CPU is free to do other work! */

DMA — Direct Memory Access

DMA controllers autonomously move data, freeing the CPU for computation. Critical for high-throughput ADC sampling, UART, SPI, and audio.

  • Circular mode enables continuous reception
  • Half-transfer and full-transfer interrupts available
  • IDLE line interrupt detects end of packet
  • Zero CPU cycles spent per transferred byte
  • Cache coherency must be managed on Cortex-M7
C — Low Power 
#include "stm32l4xx_hal.h"

/* Configure RTC wakeup every 10 s */
HAL_RTCEx_SetWakeUpTimer_IT(
    &hrtc, 10, RTC_WAKEUPCLOCK_CK_SPRE_16BITS);

void enterStopMode(void) {
    /* Disable all non-essential clocks */
    HAL_SuspendTick();

    /* Enter STOP2 — SRAM retained, ~1 µA */
    HAL_PWREx_EnterSTOP2Mode(
        PWR_STOPENTRY_WFI);

    /* MCU resumes here after wakeup */
    SystemClock_Config(); /* re-init PLL */
    HAL_ResumeTick();
}

/* Wakeup ISR */
void RTC_WKUP_IRQHandler(void) {
    HAL_RTCEx_WakeUpTimerIRQHandler(&hrtc);
}

Ultra Low Power Design

Modern MCUs offer multiple sleep states with different wake-up latencies and current draws, enabling battery life measured in years.

  • STOP2 on STM32L4: ~1 µA with RAM retention
  • Standby mode: sub-µA but RAM lost
  • Always wake via RTC, WKUP pin, or LPUART
  • Disable or clock-gate unused peripherals first
  • Reinitialise PLL after stop mode returns
C — Watchdog 
/* Independent Watchdog — runs on LSI ~32 kHz */
IWDG->KR  = 0xCCCC; /* Start WDT */
IWDG->KR  = 0x5555; /* Unlock registers */
IWDG->PR  = 4;      /* Prescaler /64 → 2 ms tick */
IWDG->RLR = 2000;   /* Reload → 4 s timeout */
IWDG->KR  = 0xAAAA; /* Reload counter */

/* "Pet" the watchdog in main loop */
void mainLoop(void) {
    while (1) {
        doWork();
        IWDG->KR = 0xAAAA; /* reset timer */
    }
    /* If loop hangs → WDT resets MCU */
}

/* Check reset cause on boot */
if (__HAL_RCC_GET_FLAG(RCC_FLAG_IWDGRST))
    logEvent("WDT reset detected");

Watchdog Timer (IWDG)

The Independent Watchdog runs on its own clock, separate from the main system clock, ensuring a reset even if the CPU clock fails.

  • IWDG is independent of the main system clock
  • Cannot be disabled after started (by design)
  • WWDG (window WDT) also detects too-early petting
  • Always check reset cause to log watchdog events
  • Essential for IEC 61508 / ISO 26262 safety compliance

Communication Protocols

Choose the right protocol for every job

🔀
UART
Universal Async Receiver-Transmitter
Asynchronous point-to-point serial. Uses TX/RX wires. No shared clock. Baud rate must match on both ends. Simple and ubiquitous.
⚡ Up to ~5 Mbps
🔁
SPI
Serial Peripheral Interface
Synchronous full-duplex. 4 wires: SCK, MOSI, MISO, CS. Master-slave topology. Fast, no addressing needed, one CS per slave.
⚡ Up to 80+ MHz
🔗
I²C
Inter-Integrated Circuit
2-wire multi-master bus. Supports up to 127 slave devices on one bus. Slower than SPI but requires only SDA + SCL.
⚡ Up to 3.4 MHz (HS)
🚌
CAN Bus
Controller Area Network
Differential multi-master bus designed for automotive and industrial. Robust error detection, priority-based arbitration, no central master.
⚡ Up to 1 Mbps (CAN FD: 8 Mbps)
🔌
USB
Universal Serial Bus
High-speed host/device protocol. Supports CDC (virtual COM), HID, MSC classes. USB FS at 12 Mbps, HS at 480 Mbps.
⚡ Up to 480 Mbps (HS)
📻
LIN Bus
Local Interconnect Network
Single-wire, low-cost sub-bus for automotive body electronics. Master-slave, 20 kbps max. Commonly combined with CAN for cost reduction.
⚡ Up to 20 kbps
🔢
1-Wire
One-Wire Protocol (Dallas)
Single-wire data + power parasite supply. Primarily used by temperature sensors like DS18B20. Very low speed but minimal wiring.
⚡ Up to 16.3 kbps
🎵
I²S
Inter-IC Sound
3-wire synchronous audio bus: SCK, WS (word select), SD. Carries PCM audio data for DACs, ADCs, and digital microphones (PDM/I²S).
⚡ Up to 3.072 MHz (audio)
